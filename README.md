# Crypto-Exam

# Общие принципы криптографии. Защита от несанкционированного доступа. Основные задачи, решаемые криптографией.

**Криптография** - это часть системы безопасности, которая разрешает доступ к чему-либо одним людям, но не разрешает другим.

### Правило слабого звена

**<p align=center>Система безопасности надежна настолько, насколько надежно ее самое слабое звено</p>**

Общие принципы криптографии включают в себя фундаментальные концепции и методы, которые обеспечивают конфиденциальность, целостность, аутентификацию и недоступность информации для несанкционированного доступа. Эти принципы являются основой для разработки криптографических систем и протоколов.

### Основные принципы криптографии:

1. **Конфиденциальность (Confidentiality)**:
   - Обеспечение того, чтобы информация была доступна только тем, кто имеет соответствующие права доступа. Шифрование используется для преобразования открытого текста в зашифрованный текст, который не может быть прочитан без ключа дешифрования.
   - Примеры: симметричное шифрование (AES), асимметричное шифрование (RSA).

2. **Целостность (Integrity)**:
   - Обеспечение того, чтобы данные не были изменены или повреждены в процессе передачи или хранения. Механизмы целостности позволяют обнаружить любые изменения данных.
   - Примеры: криптографические хеш-функции (SHA-256), коды аутентификации сообщений (HMAC).

3. **Аутентификация (Authentication)**:
   - Проверка подлинности участников коммуникации и данных. Аутентификация гарантирует, что данные поступают от доверенного источника.
   - Примеры: цифровые подписи (DSA, RSA), протоколы аутентификации (Kerberos).

4. **Недоступность для отказа (Non-repudiation)**:
   - Обеспечение невозможности отказа от отправки сообщения. Этот принцип гарантирует, что отправитель не сможет отвергнуть факт отправки сообщения.
   - Примеры: цифровые подписи с временными метками.

5. **Устойчивость к атакам (Attack Resistance)**:
   - Разработка криптографических систем, устойчивых к различным видам атак, включая грубую силу, криптоанализ и атаки на основе утечки информации.
   - Примеры: использование длинных ключей, применения соли в хешировании паролей.

+ `Открытый (исходный) текст` — данные (не обязательно текстовые), передаваемые без использования криптографии или другими словами незашифрованные данные.
+ `Шифротекст, шифрованный (закрытый) текст` — данные, полученные после применения криптосистемы (обычно — с некоторым указанным ключом). Другое название: криптограмма.
+ `Шифр, криптосистема` — семейство обратимых преобразований открытого текста в шифрованный.
+ `Ключ` — параметр шифра, определяющий выбор конкретного преобразования данного текста. В современных шифрах криптографическая стойкость шифра целиком определяется секретностью ключа (принцип Керкгоффса). Также выделяют ключ шифрования (encryption key) и ключ расшифрования (decryption key)
+ `Шифрование` — процесс нормального применения криптографического преобразования открытого текста на основе алгоритма и ключа, в результате которого возникает шифрованный текст.
+ `Расшифровывание` — процесс нормального применения криптографического преобразования шифрованного текста в открытый.
+ `Асимметричный шифр, двухключевой шифр, шифр с открытым ключом` — шифр, в котором используются два ключа, шифрующий и расшифровывающий. При этом, зная лишь ключ зашифровывания, нельзя расшифровать сообщение, и наоборот.
+ `Открытый ключ` — тот из двух ключей асимметричной системы, который свободно распространяется. Шифрующий для секретной переписки и расшифровывающий — для электронной подписи.
+ `Секретный ключ, закрытый ключ` — тот из двух ключей асимметричной системы, который хранится в секрете.
+ `Криптоанализ` — наука, изучающая математические методы нарушения конфиденциальности и целостности информации.

### Классическая задача
```
Алиса <-------------------------------------------> Боб
             m - открытый текст (секретный)
                            ^
                            |
                            |
                            |
                           Ева
```

`Ева (eavesdropper) ` - противник, пассивный злоумышленник (подслушивающий). Она может прослушивать сообщения между Алисой и Бобом, но она не может влиять на них.

<br>

Как защитить канал от прослушки Евы:
1. секретный канал могут создать Алиса и Боб, чтобы избежать прослушки
2. **Стеганография** - наука, изучающая способы сокрытия факта передачи данных
3. шифрование - преобразование информации от несанкционированного доступа (НСД) третьих лиц. 
   
Шифрование $\not=$ Кодирование. Шифрование $\subset$ Кодирование

### Симметричное шифрование

способ шифрования, в котором для шифрования и расшифрования применяется один и тот же криптографический ключ.

```
Симметричное шифрование

Алиса <-------------------------------------------> Боб
             m - открытый текст (секретный)
                            ^
                            |
                            |
                            |
                           Ева
          c                                   с
         / \                                 / \
        /   \                               /   \
Алиса <-------------------------------------------> Боб
    к - ключ                                        к
                организуют секретный канал          |
                                                    |
                                                    m
```

c = E(k, m)-  шифр текст

m = D(k, c) - расшифрование, m - открытый текст

### Симметричное шифрование
```
Симметричное шифрование 

Алиса <-------------------------------------------> Боб
    m - сообщение  открытый канал (ок)        |     PKБ
                            ^             ____|     SKБ
                            |            |   
                            |            | 
                           Ева           | 
                            |            |   
                            |            |
      <-----------------------------------          PKБ
                    аутентичный канал    |
                    C = E(PKБ, m)        |
                                         |
            c = E(PKБ, m)          m = D(SKБ, c)   
```

PK - public key

SK - secure key

Алиса дешифрует секрет
Идут парой и являются результатом одного алгоритма G
аутентичный канал - находятся данные, которые все видят read only

`Схема аутентификации`

c = E(SKБ, m)

m = D(PKБ, c)

## Приницип Кирхгофа

**<p align=center>Надежность любой  схемы шифрования должна зависеть зависеть только от ключа и не должна зависеть от алгоритмов шифрования и расшифрования.</p>**

Принцип Кирхгофа в криптографии, также известный как принцип Кирхгофа-Шеннона или аксиома Кирхгофа, гласит, что безопасность криптосистемы должна основываться на секретности ключа, а не на секретности алгоритма. Этот принцип был сформулирован в 1883 году голландским криптографом Августом Кирхгофом и до сих пор является основополагающим в современной криптографии.

### Основные положения принципа Кирхгофа:

1. **Открытость алгоритма**: Алгоритм шифрования и дешифрования должен быть известен или предполагаться известным. Вся безопасность системы должна полагаться только на секретность ключа.
2. **Безопасность ключа**: Ключ должен быть секретным и надежно защищенным. Без знания ключа невозможно дешифровать сообщение.
3. **Устойчивость к взлому**: Даже если злоумышленник знает алгоритм, ему должно быть практически невозможно расшифровать сообщение без знания ключа.

### Причины следования принципу Кирхгофа:

- **Анализ и проверка безопасности**: Открытый алгоритм можно широко анализировать и тестировать на уязвимости сообществом криптографов, что повышает доверие к его безопасности.
- **Универсальность и совместимость**: Открытые алгоритмы позволяют создавать совместимые системы и стандарты, которые могут использоваться различными организациями и устройствами.
- **Защита от утечек**: В случае, если алгоритм случайно станет известен, система все равно останется безопасной, если ключ остается секретным.

### Примеры применения:

- **AES (Advanced Encryption Standard)**: Алгоритм шифрования AES является открытым и общедоступным, и его безопасность основана на секретности ключа.
- **RSA (Rivest-Shamir-Adleman)**: Криптосистема RSA также основывается на открытом алгоритме, безопасность которого обеспечивается секретностью частного ключа.

Принцип Кирхгофа противопоставляется подходу "безопасности через скрытность" (security through obscurity), который полагается на секретность самого алгоритма. Современная криптография придерживается принципа Кирхгофа, так как он обеспечивает более устойчивую и проверяемую безопасность.

## Задачи криптографии

1. Шифрование 
   1. защита от НСД
   2. Хранение данных
2. Обеспечение целостности
   1. (Хэширование)
3. Электронная цифровая подпись (inside PKI *Public Key Infostructure*)
   1. Аутентификация
4. Доказательство с нулевым разглашением
5. Электронное голосование ( + задача вычисляется массово, на нее нельзя повлиять, результаты такого голосования будут корректны)
   1. Аукцион
6. Block-chain (доверенная последовательность цепочек каких-либо действий)
7. Неотслеживаемость (шифрование данныех, dns серверов) (массовая (onion-server) или бытовая (банк))
8. Защита от отказа приема сообщения и защита от отказа создания сообщения
9. Доверенные вычисления
